
# Badge Design — `.wt-type-tag`

エクササイズ行の先頭に表示される、種目タイプを示す小さなバッジの配色ルール。

---

## 1. バッジ背景色のルール

```
background: oklch(0.32 0.04 <hue>)
```

| パラメータ | 値 | 意図 |
| --- | --- | --- |
| 明度 (L) | `0.32` | 暗いObsidianのUIベースに馴染む低明度 |
| 彩度 (C) | `0.04` | 強すぎないミュート寄せ。隣同士の色相差を出しつつ主張しすぎない |
| 色相 (H) | メニュー名から算出 | 同じメニューは常に同じ色・違うメニューは違う色 |

色相はメニュー名の文字列から決定論的に算出する。タイプ（`sets` / `emom` / `cardio`）ではなく**メニュー名単位**で色が変わる点に注意。

---

## 2. 文字色のルール

```
color: oklch(0.85 0.06 <hue>)
```

| パラメータ | 値 | 意図 |
| --- | --- | --- |
| 明度 (L) | `0.85` | 暗い背景に対して高コントラスト |
| 彩度 (C) | `0.06` | 背景よりわずかに鮮やかにし、文字側に主役感を出す |
| 色相 (H) | 背景と同じ hue | 背景と同色相のペアカラー |

メニューに固定色が設定されている場合は、その色を文字色として直接使用する。背景は固定色を `--wt-surface-2` と 30:70 でブレンドした値になる。

---

## 3. 参考カラーパレット（6色）

動的生成の色相とは別に、固定パレットでカテゴリ分けする場合の候補。現状は実装に組み込んでおらず、将来 `category` などを導入した際に利用予定。

| ID | 色名 | Hex | 特徴 |
| --- | --- | --- | --- |
| warm-1 | Cocoa | `#452C24` | 暖色・赤茶。ベース色 |
| warm-2 | Umber | `#4A3520` | 暖色・橙茶。warm-1 から橙寄りにシフト |
| cool-1 | Navy | `#2B3347` | 寒色・深青。ベース色 |
| cool-2 | Indigo | `#332B47` | 寒色・青紫。cool-1 から紫寄りにシフト |
| green-1 | Forest | `#293723` | 緑系・深緑。ベース色 |
| green-2 | Teal | `#1F3530` | 緑系・青緑。green-1 から青緑寄りにシフト |

6#293723色は warm / cool / green の3ファミリーに2色ずつ対応しており、各ファミリーは同明度帯でそれぞれ hue を約15–20° ずらした2色で構成される。隣接ファミリー間の色相差は90°以上確保している。
