# Obsidian Workout Tracker — Requirements

## Overview

ワークアウトの記録を種目ごとに正規化された構造データとして保存し、スマホから快適に入力できるObsidianプラグイン。専用のダッシュボードファイルをリッチUIとして表示し、各日のワークアウトを個別の`.md`ファイル（YAMLフロントマター）で管理する。

## Problem Statement

これまでdaily noteに自然言語でワークアウトを記録していたため、記載粒度にばらつきがあり、種目ごとのフォーマットも統一されていなかった。その結果、データの解析や振り返りが困難だった。

## Scope

- **User**: 個人利用のみ（シングルボルト、シングルユーザー）
- **Distribution**: 公開・配布は対象外
- **Platform**: モバイル（スマホのObsidian）を主な利用環境とする
- **Phase**: 最小限の動作するMVPを優先する

## Functional Requirements

### Core Features

- **[FR-01]** `dashboard.md` をカスタムビューとして登録し、専用のリッチUIで表示する
- **[FR-02]** ダッシュボードに「Add」ボタンを表示し、タップで種目選択ダイアログを開く
- **[FR-03]** 種目選択後、その種目タイプに応じた入力UIを表示してワークアウトを記録する
- **[FR-04]** 記録したデータを `workout/YYYY-MM-DD.md`（YAMLフロントマター形式）に保存する
- **[FR-05]** ダッシュボードに直近5件のワークアウトをカンバン風の縦1列リストで表示する
- **[FR-06]** 種目は3つのタイプに分類し、タイプごとに異なる入力UIを提供する
- **[FR-07]** プラグイン設定画面で種目の追加・編集・削除ができる

### Exercise Types

| タイプ | 入力フィールド | 例 |
|--------|--------------|-----|
| `sets` | セットごとにreps（複数セット追加可能） | 懸垂、斜め懸垂、腹筋、スクワット |
| `emom` | reps × sets（2フィールド） | 腕立てemom、ディップス、懸垂emom |
| `cardio` | commentのみ | 有酸素（ダッシュ、水泳など） |

各種目にはcommentフィールドを持つ（任意入力）。

### Data Schema

各日のワークアウトファイル（`workout/YYYY-MM-DD.md`）のYAMLフロントマター：

```yaml
---
date: YYYY-MM-DD
exercises:
  - menu: 懸垂
    type: sets
    sets: [12, 7]
    comment: ""
  - menu: 腕立てemom
    type: emom
    reps: 9
    sets: 10
    comment: ""
  - menu: 有酸素
    type: cardio
    comment: "ダッシュ"
---
```

### UI Components

- **Commands**: なし（ダッシュボードファイルを開くことで起動）
- **Ribbon**: なし（MVPスコープ外）
- **Views / Panels**: `dashboard.md` に紐づくカスタムビュー（`WorkoutDashboardView`）
- **Modals**:
  - 種目選択ダイアログ（Addボタンタップ時）
  - 種目タイプ別の入力フォームモーダル
- **Settings Tab**: 種目リストの管理（種目名・タイプの追加・編集・削除）

### Dashboard Card Display

カンバン風リストの各カードに以下を表示する：

- date（記録日）
- 種目名
- 回数（setsタイプ: セットごとのreps一覧、emomタイプ: reps × sets、cardioタイプ: 表示なし）
- comment（入力されている場合）

#### UIイメージ

```
┌─────────────────────────────────────────────┐
│  Workout Log                                │
│                                             │
│  [ + Add Workout ]                          │
│                                             │
│  2026-04-28                                 │
│  ─────────────────────────────────────────  │
│  │ 懸垂          ×12  ×7                   │
│  │ 腕立てemom    9 × 10                    │
│  │ 有酸素        ダッシュ                  │
│                                             │
│  2026-04-27                                 │
│  ─────────────────────────────────────────  │
│  │ 腹筋          ×10  ×10  ×10            │
│  │ スクワット    ×23                       │
│  │   > 手首が少し痛い                      │
│                                             │
│  2026-04-25                                 │
│  ─────────────────────────────────────────  │
│  │ 懸垂emom      5 × 8                     │
│  │ 有酸素        ダッシュ                  │
│                                             │
│  （直近5件まで表示）                        │
│                                             │
└─────────────────────────────────────────────┘
```

- 左端の縦線（`│`）は種目の区切りを示す左ボーダー
- repsはバッジ風に `×12` と表示
- emomは `reps × sets` 形式で1行表示
- commentは `>` インデントで種目の下に表示

### Data Access

- **Reads**: `workout/YYYY-MM-DD.md` のYAMLフロントマター（直近5件分）
- **Writes**: `workout/YYYY-MM-DD.md` の新規作成・同日分への上書き追記
- **Persistence**: 種目リスト定義（プラグイン設定として`data.json`に保存）

## Non-Functional Requirements

- **[NFR-01]** 個人利用のためi18nや多ユーザー対応は不要
- **[NFR-02]** モバイルObsidianで動作すること（タップ操作で完結するUI）
- **[NFR-03]** 標準の`.md`ファイル形式で保存するためObsidian Syncや他の同期サービスと互換性を保つ
- **[NFR-04]** MVPとして最小限の機能を優先し、複雑な処理は避ける

## Out of Scope

- 統計・グラフ表示（種目ごとのreps推移など）
- 通知・リマインダー機能
- データエクスポート（CSV/JSONダウンロード）
- リボンアイコンの追加
- daily noteへの書き込み連携

## Decision Log

- **カンバンカード表示項目**: date・種目・回数・comment をすべて表示する
- **同日の記録**: 1日1ファイル（`workout/YYYY-MM-DD.md`）に上書き追記する
- **種目プリセット**: なし。設定画面から空の状態で始め、ユーザーが追加する
